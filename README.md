# 🩺 HealthBot: AI-Powered Patient Education System (Web Edition)

> A decoupled, modern web-based patient education chatbot. It features a **FastAPI backend** running a stateless **LangGraph** & **LangChain** agent workflow, powered by **Tavily Search** and **OpenRouter**, combined with a premium, high-contrast dark themed **React/Vite frontend**.

HealthBot is a prototype developed for **MediTech Solutions** as part of a Capstone Project in Healthcare AI. The system addresses a critical gap in patient education by providing:

- 🔍 **Real-time health information** fetched from reputable web sources via Tavily Search
- 📝 **Patient-friendly summaries** generated by an LLM (no medical jargon)
- 🧠 **Comprehension quizzes** to assess patient understanding
- 🏆 **Graded feedback with citations** from the health information summary
- 🔄 **Multi-topic sessions** with full state reset to maintain patient privacy

## 📖 Overview

HealthBot addresses critical gaps in patient education by providing:
* 🔍 **Real-time health information** retrieved from reputable web sources using Tavily Search.
* 📝 **Patient-friendly summaries** generated by an LLM with layperson terminology (medical jargon explained in parentheses).

## 🔄 Stateful Workflow & API Routing

To run a multi-stage conversational graph over stateless HTTP API requests, the backend employs two production patterns:
1. **Idempotence Checks**: Nodes like `search_tavily` and `summarize_results` inspect the incoming state. If their corresponding properties are already populated, they instantly return `{}` to bypass duplicate LLM and search tool calls.
2. **Execution Yielding**: The graph uses conditional edges (`should_start_quiz` and `should_evaluate`) to yield control by returning `END` if the user is not ready or has not yet typed an answer, returning the current state package back to the frontend.

```text
               START
                 │
                 ▼
[api/start]  get_topic         → Reset state and initialize topic
                 │
                 ▼
             search_tavily     → (Idempotent) Fetch web sources
                 │
                 ▼
           summarize_results   → (Idempotent) Generate layperson summary
                 │
                 ▼
            wait_for_ready     → Check readiness
                 │
                 ▼ [ready_for_quiz != "yes"]
                END (Yield to React) ──► Show summary to patient
                 │
                 ▼ [ready_for_quiz == "yes"]
[api/submit-  generate_quiz    → (Idempotent) Generate comprehension question
 readiness]      │
                 ▼
             collect_answer    → Load quiz answer input
                 │
                 ▼ [user_answer is empty]
                END (Yield to React) ──► Prompt user for answer
                 │
                 ▼ [user_answer is filled]
[api/submit- evaluate_answer   → Grade (A-F) with citations
  answer]        │
                 ▼
             display_grade     → Log evaluation grade
                 │
                 ▼
            restart_or_exit    → Route to restart or exit
                 │
                 ▼
                END (Yield to React) ──► Display graded feedback dashboard
```

## ⚙️ Environment Setup

### 1. API Configuration

Create a file named `.env` in the `backend/` directory:

```env
OPENAI_API_KEY="sk-or-v1-your-openrouter-key-here"
TAVILY_API_KEY="tvly-your-tavily-key-here"
OPENAI_MODEL="google/gemini-2.5-flash"
```

## 🚀 Running the Application

### 1. Start the FastAPI Backend

1. Navigate to the backend directory:
   ```powershell
   cd backend
   ```
2. Activate your virtual environment:
   ```powershell
   # Windows (PowerShell)
   ..\.venv\Scripts\Activate
   ```
3. Launch the FastAPI server:
   ```powershell
   python -m uvicorn main:app --reload
   ```
   The backend API will be available at [http://127.0.0.1:8000](http://127.0.0.1:8000) with Swagger documentation at `/docs`.

### 2. Start the React Frontend

1. Open a new terminal and navigate to the frontend directory:
   ```powershell
   cd frontend
   ```
2. Install npm dependencies:
   ```powershell
   npm install
   ```
3. Start the Vite development server:
   ```powershell
   npm run dev
   ```
4. Open your browser and navigate to [http://localhost:5173/](http://localhost:5173/) to interact with the system!
